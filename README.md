# 空间锚点 · 英语词感地图

## 你学英语的痛苦，根源不在你

我英语专业八级，还读了英语研究生，可照样说不出英语——或者说，说不出正确、体面的英语。

这可能是我们这辈子最糟糕的 ROI 了。我猜，很多人跟我一样。在苦海中沉浮久矣，终见一丝微光。乃作此册，以飨同路人。

与君共勉。愿有所益。

**中国人和英语之间，隔着一层翻译。这层翻译，就是罪魁祸首。**

我们的大脑里装了一套"高语境、模糊化、默认对方懂我"的中文沟通系统，却试图用它来运行一套"低语境、精准化、默认对方不懂我"的语言。说英语时先想中文、再翻译——反应慢、介词乱用、时态纠结。不是词汇量不够，是两套系统在打架。

**解决方案不是更努力，是建第二套独立的语言系统——直接看画面，直接理解，直接说。不翻译。**

---

## 建这套系统，从空间框架开始

英语是一门空间语言。它的底层是一套空间关系：什么东西在上面、在下面、在里面、在外面、在靠近、在远离。

**小品词（on/off/up/down/in/out）就是这套空间框架。** 没有这个框架，房子立不起来。

**常用动词（put/get/take/give/pull/cut）是砖块。** 同一块砖放在框架的不同位置，功能完全不一样：

```
put on       = 砖放在"上面"      → 穿上 / 上演 / 打开
put in       = 砖放在"里面"      → 放进去 / 投入
put off      = 砖从原位移"开"    → 推迟 / 反感
put up with  = 撑在"高处"承受    → 忍受
```

动词决定"做什么"，小品词决定"做的方向/结果/状态"。两者组合，就能表达世界上几乎所有场景。

**但大多数人怎么学的？跳过地基，跳过框架，直接背"家具的名字"——背了忘，忘了背。**

## 这个项目做什么

用一个物理画面当锚点，推导出一个词所有的抽象用法。

每个词我们找到它最本质的物理画面（on = 杯子放在桌面上），然后一步步推演出所有抽象含义（on the phone / on a diet / count on me / a book on history——全来自同一幅画面）。

**读者看完不是"记住了一个词"，而是体验到了它的空间逻辑。以后在任何场景遇到，自己能推出来。**

结果是一套**可视化空间词典**：每个词一个独立 HTML 页面，用 SVG 动画画出核心物理画面，用小白框推演到所有抽象含义。共 74 个单词深挖页 + 10 个词族对比页。

## 核心方法：三句话

**第一句：理解核心画面**

| 词 | 物理画面 | 一句话 |
|----|---------|--------|
| **ON** | 杯子放在桌面上 | 接触+支撑 |
| **IN** | 物体被边界包围 | 包围=IN |
| **OFF** | 物体从表面脱离 | 脱离接触 |
| **UP** | 从低到高移动 | 朝上 |
| **SNAP** | 绷紧后突然断裂 | 极限后断裂 |

**第二句：从画面推导抽象**

SNAP 的例子：
- 精神崩溃 🧠 承受力像橡皮筋，到极限就断 → I snapped at him
- 拍照 🧠 老式快门"咔嚓"声像断裂声 → Snap a photo
- 快速决定 🧠 像断裂一样"瞬间完成" → a snap decision

**第三句：不翻译，直接看画面**

```
"The engine died."     → 引擎像生命体，自己停止运转
"The idea came to me." → 主意像客人走到我面前
"He's on the phone."   → 注意力附着在电话线上
```

翻译是把一次空间体验变成一堆抽象概念——恰恰毁掉了英语的底层逻辑。
